SCC Jupyter setup

This tutorial will show you how to run Jupyter notebook in scc, which you can access from a browser on your local machine

Prerequisites

I assume here you have Python 3 and a virtual python environment for your project setup.

Login to SCC

$ ssh <user>@scc1.bu.edu

Request a GPU

$ qrsh -l gpus=1 -l gpu_c=3.5

You may skip this step if you do not need a GPU (not training Machine Learning models)

Your shell would show now as

[<user>@scc-c13 ~] $

Note that "scc-c13" may be different for you. Note the name here.

Go to your project directory, and activate python environment (here: environment name is 'env')

$ cd <project-dir>
    $ source env/bin/activate

Start jupyter notebook

jupyter notebook --no-browser

In the output, you'll see something like:

[I 10:49:35.284 NotebookApp] The Jupyter Notebook is running at:
    [I 10:49:35.284 NotebookApp] http://scc-c13:8888/

Note that the port here is: 8888

To view the jupyter notebook in your local browser

We want to forward the port of scc on which Jupyter is running to our local machine

Now, from another terminal, run the following command 

$ ssh <user>@scc1.bu.edu -L 8002:scc-c13:8888

On SCC, Jupyter is running on scc-c13:8888, On our local machine we want to run it on port 8002,

On a browser on your local machine, go to

http://localhost:8002/

You should see the jupyter notebook Here.

Congratulations for reaching the end!

Have fun training models now.

References:

http://scv.bu.edu/examples/ML/jupyter_notebook_gpu/ https://www.digitalocean.com/community/tutorials/how-to-install-run-connect-to-jupyter-notebook-on-remote-server

Note if you are training Neural Nets on Tensorflow

Tensorflow will use all available cores, and SCC will kill your process, if you use more resources than you requested. The below is a workaround for this in jupyter notebook:

# import tensorflow as tf
    
    session_conf = tf.compat.v1.ConfigProto(
          intra_op_parallelism_threads=4,
          inter_op_parallelism_threads=4)
    sess = tf.compat.v1.Session(config=session_conf)
    
    tf.compat.v1.keras.backend.set_session(sess)

Here, you describe how many CPU threads your program uses. THis isn't the best way. You can try playing around with different values.
